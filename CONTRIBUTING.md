# Contributing to Local Gallery

We love your input! We want to make contributing to Local Gallery as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. If you've changed APIs, update the documentation
4. Ensure the test suite passes
5. Make sure your code lints
6. Issue that pull request!

## Code Style

- Use TypeScript for all frontend code
- Follow Rust conventions for backend code
- Use Prettier for code formatting
- Follow ESLint rules

## Testing

- Write unit tests for new functionality
- Add E2E tests for user-facing features
- Ensure all tests pass before submitting PR

## Reporting Bugs

Use GitHub issues to report bugs. Include:

- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Environment details (OS, version)

## Feature Requests

Use GitHub issues with the `enhancement` label. Include:

- Clear description of the feature
- Use case and motivation
- Proposed implementation (if any)
